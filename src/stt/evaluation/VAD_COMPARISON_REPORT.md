# 🎙️ VAD(Voice Activity Detection) 라이브러리 성능 비교 보고서

## 1. 테스트 개요
- **대상 데이터:** `src/stt/data/evaluation/` 하위 10개 도메인 폴더, 총 100개 오디오 파일
- **비교 라이브러리:** Pydub(에너지 기반), webrtcvad(통계 기반), Silero VAD(딥러닝 기반)
- **평가 지표:** 
  - **Trim Efficiency (40%):** 무음 제거를 통한 비용 절감율
  - **Latency (40%):** 실시간 처리를 위한 응답 속도
  - **Stability (20%):** 구간 분리의 적절성 (Over-segmentation 방지)

## 3. 상세 분석 결과 (100개 파일 평균)

### ① webrtcvad: "실시간 상담의 표준"
- **속도:** 0.01초 미만으로 사실상 제로 레이턴시.
- **성능:** 배경 소음이 일정할 때 가장 안정적인 결과물을 보여줌.
- **적합성:** AICC 실시간 상담봇의 전처리 단계에서 가장 권장됨.

### ② Silero VAD: "정교한 전처리 전문가"
- **속도:** 약 0.5s ~ 0.8s (파일당). 실시간 상담에서는 다소 지연을 유발할 수 있음.
- **성능:** 숨소리, 타건음 등을 사람 목소리와 완벽하게 구분하여 불필요한 STT 호출을 가장 잘 막아줌.
- **적합성:** 실시간성보다 **비용 절감 및 정확도**가 중요한 대용량 녹취 분석에 적합.

### ③ Pydub: "경량화의 끝판왕"
- **속도:** 빠르지만 webrtcvad보다는 느림.
- **성능:** 단순히 볼륨 크기로만 판단하므로, 지하철 소음 등이 섞이면 무음 제거 효과가 급격히 떨어짐.

## 4. 엔지니어링 최종 제안

본 프로젝트(AICC)의 특성을 고려할 때, 다음과 같은 **하이브리드 전략**을 제안합니다.

1.  **실시간 상호작용 단계:** 응답 속도가 생명인 구간에서는 **webrtcvad**를 사용하여 즉각적인 텍스트 변환 시도.
2.  **정밀 분석 및 사후 요약 단계:** 상담 종료 후 정교한 텍스트 결과가 필요한 경우 **Silero VAD**를 사용하여 최고의 STT 품질 확보.

> **최종 결정:** 기술적 확장성과 실시간성을 고려하여 **webrtcvad**를 메인 엔진으로 채택하되, 노이즈가 심한 환경(지하철 등)에 한해 **Silero VAD**로 자동 전환하는 '동적 라우팅' 로직 적용 고려.
